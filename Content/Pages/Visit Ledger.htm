Visit Ledger

This tab will display all Transaction information regarding the Visit.  This will break down the Billing, Payments and other information for this visit.

  • Double-click the visit for view on the Tree View on the left.
  • Click the patient name/visit hyperlink to open the visit in the Registration form (outlined in red below).
  • The Current Payer section is where the status can be viewed for each payer associated with the visit and change the Current Payer to Self Pay.

To change the current payer to a previously-billed insurance or to Self Pay, check the box of the ‘Current Payer’ to switch to.  A pop up will appear to confirm the change.  Click "Yes" to confirm, then click Save.

 


 

Note:  If billing has not occurred for the secondary payer, setting to Current will NOT produce a claim.  See Secondary Billing for guidance in this process.

  • The CPT Line Item grid will show all of the CPT Codes billed for this visit.

  • If line item posting was performed in Payments And Adjustments, the line item details will appear in this grid. They can also be edited or entered here if this was not done in Payments And Adjustments.
  • Patient Resp Amount can be entered to indicate how much the patient owes for the visit. If Statement Trigger is set to “Immediately bill patient responsibility amount”, the amount entered here will trigger a statement even though the total outstanding balance is still reflected as due from insurance.

NOTE: Once the balance on the visit has been transferred to SELF PAY FOR BILLING, that balance will override any amount remaining in the Patient Resp Amount column.

  • The Transaction History grid will show any posted/unposted transactions associated with the visit.

- Selecting the blue hyper linked transaction number, will open that transaction in the Charge Entry or the Payments and Adjustments screen.

Note:  There's a very handy visit transaction summary provided on this screen; it's a great tool for determining total financial activity on the account, as well as delinquency and self-pay date at a glance.

 

 

Self Pay Date:  Note the Self Pay Date on the ledger.  This is the FIRST TIME the account was set to Self Pay status.  All accounts start with Self as the current payer, until the charges are created and posted.  The first time you change the current payer to Self Pay after initial billing, the system will make a note of the date and fill it in here.  It will not update again.

Undo Visit Balance: if a user is granted permissions to this functionality, it will reverse ALL transactions and mark the account balance as 0. This will allow the insurances to be easily changed for the patient.

Note: ALL Transactions, including self pay, will be reversed from the account. Also, ALL reversals will automatically default to the current date as the transaction date. However, if an account has a $0 balance, the "Undo Visit Transactions" will become disabled.
