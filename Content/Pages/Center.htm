Center

Administration > Configuration > Center Configuration

NOTE: Fields that are purple or grayed out will require assistance from Pathways Support to change.

  • E-mail: Enter the primary e-mail address for the center.
  • User Profile Email: Enter the e-mail address of the primary security administrator. When new users are added or passwords are reset, this is the e-mail address from which the notification e-mails will be generated.
  • Web Site: Enter the URL for the center's website.

 

Accounting:

  • Tax ID: Enter the center's tax ID number.
  • Default Self Pay ID: Defaults to 1. (The moment a case is scheduled Pathways assumes there will be charges that will be due from someone. Until another payer is entered, the system assumes that the patient will be responsible for those charges. Consequently, the Self-Pay Payer ID defaults to 1.)
  • Account Number Format:
    • 1 - Same Account for each visit - all patient visits will be billed under a single Account Number.
    • 2 - New Account for each visit - each date of service will have a unique billing Account Number.
    • 3 - Patient ID - Account number will be the same as the Patient ID.
  • Statment Payment Address: Addresses are entered on the Address tab of Center Configuration and then can be selected from the drop down specific to various addresses (statements, shipping, billing, claims, etc.) Once addresses have been created, select the address that should appear on patient statements from the dropdown.

  • Allow On Screen Posting: Checking this box will enable users with the appropriate permissions to post charges and payments directly from the Charge Entry and Payment And Adjustments forms rather than having to navigate to Transaction Posting.
  • Separate Coding/Charges: When checked, the Code Entry andCoded Charge Entry functions will be two separate forms. This is commonly utilized when the coding is performed by a CBO or an outside agency.

Financial Aging is how aging calculations will be performed for reporting purposes:

  • By Visit Date (default) - Aging will be based upon the Visit Date (date of service).
  • By Self-Pay Date - Aging will be based upon the date the balance is transferred to Self-Pay.

Statement Aging is how aging calculations will be performed for delinquency statements (statements that reflect the age of outstanding balances):

  • By Visit Date - Aging will be based upon the Visit Date (date of service).
  • By Self-Pay Date (default) - Aging will be based upon the date the balance is transferred to Self-Pay.

Regulatory Reports

The Regulatory Reporting grid allows centers to add or delete the Report Codes specific to required State Reporting for their location.  

Payment Plan Configuration

Centers that choose to utilize the Payment Plan Tab in the Patient Account will need to activate the plan here and set up the desired parameters specific to center policies and procedures.

  • Enable Payment Plans: To activate the Payment Plan tab in the Patient Account, this box must be checked. The tab will not be visible if not activated.
  • Minimum Payment Amount: Centers can set a required minimum payment amount. If users attempt to set up a payment plan with a monthly payment that is less than this amount, an error ball will be invoked, preventing them from proceeding.
  • Maximum # of Payments: Centers can set a maximum number of payments. If users attempt to set up a payment plan with more payments than this amount, an error ball will be invoked, preventing them from proceeding.
  • Minimum Total Patient Balance: Centers can set a minimum Beginning Plan Balance. If users attempt to set up a payment plan with a beginning balance that is less than the amount entered here, an error ball will be invoked, preventing them from proceeding.
  • Maximum Total Patient Balance: Centers can set a maximum Beginning Plan Balance. If users attempt to set up a payment plan with a beginning balance that is greater than the amount entered here, an error ball will be invoked, preventing them from proceeding.
